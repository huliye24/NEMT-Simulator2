# 策略演化规则 (EVOLUTION_RULES)

> 最后更新: 2026-04-19

## 一、淘汰规则

### 硬性淘汰（立即执行）

| 条件 | 动作 | 备注 |
|------|------|------|
| 评分 < 20 | → DEAD | 强制淘汰 |
| 最大回撤 > 30% | → DEAD | 强制淘汰 |
| 连续亏损 > 10天 | → DEAD | 强制淘汰 |

### 软性淘汰（观察后执行）

| 条件 | 动作 | 备注 |
|------|------|------|
| 评分 < 30 | → DORMANT | 降低50%权重 |
| 夏普比率 < 0.5 (持续30天) | → DORMANT | 降低权重 |
| 回撤 > 20% | → DORMANT | 降低50%权重 |
| 与其他策略相关性 > 0.9 | → DORMANT | 保留高评分者 |

## 二、激活规则

| 条件 | 动作 |
|------|------|
| 休眠策略评分 > 50 | → ALIVE |
| 测试策略夏普 > 1.0 | → ALIVE |

## 三、测试策略升级规则

| 条件 | 从 TESTING → ALIVE |
|------|---------------------|
| 回测夏普 | > 1.0 |
| 回测胜率 | > 50% |
| 最小交易数 | > 50笔 |
| 最大回撤 | < 15% |

## 四、权重分配规则

### 等权重模式
- 所有活跃策略平均分配 100% / N

### 评分权重模式
```
权重 = (策略评分 / 所有活跃策略评分总和) × 100%
```

### 最小权重
- 任何活跃策略最小权重: 5%
- 休眠策略权重: 0%

## 六、执行频率

- 淘汰检查: 每日
- 权重重分配: 每周一
- 策略升级评估: 每月
