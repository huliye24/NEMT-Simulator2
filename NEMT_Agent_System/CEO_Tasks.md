# CEO/Founder 任务清单

> 角色: Founder (兼任 CEO)
> 职责: 维护愿景、对齐检查、最终裁决

---

## 每日任务 (Daily)

### 每日站会 (9:00 AM)
- [ ] 查看 Task Inbox 中的"我的任务"视图
- [ ] 在 Weekly Vision Sync 评论区发一条：
  - 今天主要处理 Task #XXX
  - 阻塞：无 / 等待 XXX 审批

### 每日裁决
- [ ] 检查是否有"需审核"状态的任务
- [ ] 对决策请求做出裁决并记录

---

## 每周任务 (Weekly)

### 每周愿景对齐会 (周一 10:00 AM)
主持15分钟会议，检查：

#### 1. 上周任务检查
- [ ] 是否有任务未勾选愿景检查？
- [ ] 标记任何偏离情况

#### 2. 决策记录检查
- [ ] 查看 Decision Log 最近一周的决策
- [ ] 是否有决策违背了 VISION.md？

#### 3. 评分与记录
- [ ] 给上周工作打分 (1-5)
- [ ] 记录纠正措施

---

## 里程碑验收任务

当 Vision Milestones 状态变为"开发中完成"时：

### 验收检查清单
- [ ] 代码是否引入了新依赖？ (CTO 查 package.json diff)
- [ ] 是否生成了新的未规划子项目？ (PM 查 Task Inbox)
- [ ] 核心原则是否受损？ (CEO 判断)

### 验收结论
- [ ] 全部通过 → 标记为"验收通过"
- [ ] 未通过 → 标记问题并退回

---

## 决策审批流程

当收到"决策请求"类型任务时：

### Step 1: 阅读背景
- [ ] 打开任务详情
- [ ] 阅读 PM/其他Agent的描述

### Step 2: 咨询CTO (如需要)
- [ ] 在任务评论区 @CTO 提问
- [ ] 等待 CTO 技术分析

### Step 3: 做出裁决
- [ ] 在 Decision Log 创建记录
- [ ] 填写愿景对齐分析
- [ ] 给出最终决定

### Step 4: 通知执行
- [ ] 更新任务状态为已完成
- [ ] 关联决策记录
- [ ] 通知相关Agent

## 收件箱视图使用

每次打开 Notion 时：
1. 点击侧边栏的"CEO 收件箱"视图
2. 查看筛选后的任务列表
3. 按以下优先级处理：
   - 🔴 紧急 (需裁决)
   - 🟡 待讨论
   - 🟢 可完成
