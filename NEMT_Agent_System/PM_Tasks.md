## 任务管理流程

### 创建新任务

当发现新需求时（如用户反馈、CTO建议）：

#### Step 1: 愿景检查
在创建任务前，先对照 VISION.md：
- [ ] 这个需求是否在边界之内？
- [ ] 是否与核心原则冲突？
- [ ] 有没有更简单的方案？

#### Step 2: 创建任务
在 Task Inbox 中创建任务：
- [ ] 任务名称：清晰描述
- [ ] 负责人：合适的Agent
- [ ] 任务类型：愿景检查/开发任务/决策请求
- [ ] 优先级：P0/P1/P2/P3
- [ ] 描述：详细说明背景和目标
- [ ] 关联决策：如已有决策记录

#### Step 3: 愿景对齐检查（内嵌）
```
- [ ] 是否在边界之外？
- [ ] 有没有更简单的方案？
- [ ] 是否需要CEO裁决？
```

### 任务状态流转
```
待认领 → 进行中 → 需审核 → 已完成
              ↓
           需裁决 (如需要)
```

## 场景处理

### 场景：收到新需求（给回测结果加折线图）

#### Step 1: 愿景检查
- [ ] 查看 VISION.md 边界
- [ ] 发现"不做 Web 前端"
- [ ] 评估：加图表是否算前端？

#### Step 2: 创建评估任务
- [ ] 创建任务："评估回测结果可视化需求"
- [ ] 负责人：PM（自己）
- [ ] 类型：愿景检查
- [ ] 描述：用户反馈想看权益曲线图

#### Step 3: 自检
- [ ] 勾选愿景检查清单：
  - [x] 是否在边界之外？ → 是（Web前端相关）
  - [x] 有没有更简单的方案？ → 有，导出 CSV 用 Excel 看
- [ ] 检查结果：不通过

#### Step 4: 转交裁决
- [ ] 状态改为"需审核"
- [ ] 负责人改为"CEO"
- [ ] 类型改为"决策请求"
