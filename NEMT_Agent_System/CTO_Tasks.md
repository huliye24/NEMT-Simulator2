## 代码审查流程

当收到代码审查请求时：

### Step 1: 阅读代码

- [ ] 打开相关 PR/提交
- [ ] 运行代码并测试

### Step 2: 检查清单

- [ ] 代码是否符合架构规范？
- [ ] 是否有安全漏洞？（如硬编码 Token）
- [ ] 是否有性能问题？
- [ ] 是否遵循 DRY/YAGNI 原则？
- [ ] 是否有硬编码？
- [ ] 是否有未处理的异常？
- [ ] 测试覆盖是否充分？

### Step 3: 反馈

- [ ] 给出具体的修改建议
- [ ] 标记阻塞性问题 vs 建议性修改
- [ ] 批准或要求修改

### Step 4: 合并

- [ ] 确认所有问题已修复
- [ ] 执行代码合并
- [ ] 更新任务状态
