# Vision Milestones - 愿景里程碑

> 灯塔与航程图，记录每个里程碑的愿景验收

---

## 数据库字段设计

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 里程碑名称 | Title | 里程碑标题 |
| 目标日期 | Date | 计划完成日期 |
| 实际日期 | Date | 实际完成日期 |
| 状态 | Select | 规划中/开发中/已完成 |
| 验收状态 | Select | 待验收/验收通过/验收失败 |
| 验收人 | Person | 负责验收的人 |
| 负责人 | Person | 负责开发的人 |
| 验收清单 | Text | 验收检查结果 |
| 验收日期 | Date | 验收完成日期 |

---

## 里程碑验收流程

### 触发条件
PM 将 Vision Milestones 状态改为"开发中完成"

### 验收动作
CEO + CTO 打开里程碑详情，执行验收清单

### 验收清单

```
## 里程碑验收清单

**里程碑**: 
**验收人**: 
**验收日期**: 

### 1. 代码检查
- [ ] 代码是否引入了新依赖？ (CTO 查 package.json diff)
- [ ] 是否有未规划的子项目？ (PM 查 Task Inbox)

### 2. 架构检查
- [ ] 是否符合架构设计？
- [ ] 是否有技术债增加？

### 3. 愿景对齐检查
- [ ] 使命对齐: ✅/❌
- [ ] 原则对齐:
  - [ ] 策略是生物体: ✅/❌
  - [ ] 系统适应市场: ✅/❌
  - [ ] 风控优先: ✅/❌
  - [ ] 厨房理论: ✅/❌
- [ ] 边界合规: ✅/❌

### 4. 功能检查
- [ ] 核心功能是否实现？
- [ ] 测试覆盖率是否达标？

### 5. 文档检查
- [ ] 文档是否更新？
- [ ] 决策记录是否归档？
```

### 验收结论

- [ ] **全部通过** → 标记"验收通过"
- [ ] **未通过** → 标记"验收失败"，退回修改

---

## 里程碑状态流转

```
规划中 → 开发中 → 开发中完成 → 验收通过/验收失败
                              ↓
                         验收失败 → 开发中 (退回)
```
