> 定期的纠偏机制，每周回顾愿景对齐情况

---

## 数据库字段设计

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 周次 | Title | 如 "2026-W16" |
| 日期 | Date | 会议日期 |
| 主持人 | Person | 会议主持 |
| 出席人 | Multi-select | 参与者 |
| 偏离记录 | Text | 本周偏离情况 |
| 评分 | Number | 愿景对齐评分 (1-5) |
| 纠正措施 | Text | 纠正行动计划 |
| 下周重点 | Text | 下周工作重点 |

## 会议议程

### 1. 上周任务检查 (5分钟)
检查 Task Inbox 中上周完成的任务：

- [ ] 有多少任务已完成？
- [ ] 有多少任务未勾选愿景检查？
- [ ] 标记任何偏离情况

### 2. 决策记录回顾 (5分钟)
查看 Decision Log 最近一周的决策：

- [ ] 最近有多少决策？
- [ ] 是否有决策违背了 VISION.md？
- [ ] 是否需要发出愿景漂移警告？

### 3. 评分与记录 (5分钟)
给上周工作打分：

- [ ] 使命对齐: __/5
- [ ] 原则遵守: __/5
- [ ] 边界合规: __/5
- [ ] 整体评分: __/5

记录纠正措施：

## 愿景漂移警告

当出现以下情况时，发出愿景漂移警告：

- [ ] 连续3次决策记录显示"为了性能牺牲架构"
- [ ] 连续2周评分低于3分
- [ ] 突然出现大量边界外的任务

### 警告格式

```
🚨 愿景漂移警告

**日期**: YYYY-MM-DD
**警告原因**: 

### 具体情况
> 

### 建议措施
> 

### 负责人: CEO
```
