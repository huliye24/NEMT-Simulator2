## 角色权限概览

| 角色 | Task Inbox | Decision Log | Vision Milestones | Weekly Sync |
|------|------------|--------------|-------------------|-------------|
| CEO | 全部可见/编辑 | 全部可见/编辑 | 全部可见/编辑 | 全部可见/编辑 |
| CTO | 全部可见/编辑 | 全部可见/编辑 | 全部可见/编辑 | 全部可见/编辑 |
| PM | 全部可见/编辑 | 全部可见/编辑 | 全部可见/编辑 | 全部可见/编辑 |

## 页面级权限

### CEO/Founder
- **权限**: 全部权限
- **视图**: CEO 收件箱 (过滤: 负责人=CEO)
- **能力**:
  - 查看/编辑所有页面
  - 创建/删除数据库记录
  - 设置自动化规则
  - 管理其他角色权限

### CTO
- **权限**: 全部权限
- **视图**: CTO 收件箱 (过滤: 负责人=CTO)
- **能力**:
  - 查看/编辑所有页面
  - 创建技术决策记录
  - 执行代码审查
  - 设置架构规范

### PM
- **权限**: 全部权限
- **视图**: PM 收件箱 (过滤: 负责人=PM)
- **能力**:
  - 查看/编辑所有页面
  - 创建/分配任务
  - 管理里程碑
  - 协调会议

### Tech Lead
- **权限**: 全部可见，任务级编辑
- **视图**: Tech Lead 收件箱 (过滤: 负责人=Tech Lead)
- **能力**:
  - 查看所有页面
  - 只能编辑自己认领的任务
  - 创建代码审查请求
  - 更新任务状态

## 视图权限

### CEO 收件箱视图
```
类型: Table
过滤: 负责人 Contains "CEO"
排序: 优先级 ASC, 创建日期 DESC
```

### CTO 收件箱视图
```
类型: Board
分组: 状态
过滤: 负责人 Contains "CTO"
```

### PM 收件箱视图
```
类型: Table
过滤: 负责人 Contains "PM" OR 状态 Contains "待认领"
排序: 优先级 ASC
```

### Tech Lead 收件箱视图
```
类型: Board
分组: 状态
过滤: 负责人 Contains "Tech Lead"
排序: 优先级 ASC, 更新日期 DESC
```

## 自动化权限

### 自动化规则执行者

| 规则 | 执行权限 |
|------|----------|
| 负责人变更通知 | 数据库所有者 |
| 决策请求升级 | 数据库所有者 |
| 代码审查通知 | 数据库所有者 |
| 周会提醒 | CEO |

## 权限继承

```
工作区 (Workspace)
    ↓
团队空间 (Teamspace) [可选]
    ↓
页面 (Page)
    ↓
数据库 (Database)
    ↓
视图 (View)
```

下层可以继承或覆盖上层权限。

## 访客邀请

如需邀请外部人员：

1. 创建访客账户
2. 邀请加入工作区 (作为 Guest)
3. 仅共享需要的页面
4. 设置为"可以查看"
