## 一、项目概述

### 1.1 项目愿景

> **一个基于"非平衡市场理论"的BTC交易系统——像物理学家研究市场**

NEMT Simulator 旨在通过非线性薛定谔方程 (NLS) 模拟市场状态，识别四相位周期 (A/B/C/D)，生成交易信号，实现量化交易的闭环验证。

### 1.2 核心技术栈

| 层级 | 技术 | 用途 |
|------|------|------|
| **前端** | React + TypeScript + Vite | Web 可视化界面 |
| **协议层** | MCP (Model Context Protocol) | 工具调用标准化 |
| **核心引擎** | Python quant-sync-server | Notion/MATLAB/Signal 集成 |
| **计算层** | MATLAB Engine / TypeScript | NEMT 数值计算 |
| **数据层** | Notion API | 策略存储与结果回写 |
| **执行层** | Go Server | 交易信号执行 |

## 八、风险与挑战

### 8.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| MATLAB 引擎兼容性 | 高 | 已实现 TypeScript 本地降级 |
| Notion API 配额限制 | 中 | 实现本地缓存机制 |
| 交易信号延迟 | 高 | 异步队列 + 重试机制 |

### 8.2 市场风险

| 风险 | 说明 | 应对 |
|------|------|------|
| 模型假设失效 | NLS 方程基于历史数据 | 多周期验证，持续迭代 |
| 相位识别错误 | 导致仓位判断失误 | 增加置信度阈值 |
| 黑天鹅事件 | 超出模型处理范围 | 硬止损 + 动态仓位 |
