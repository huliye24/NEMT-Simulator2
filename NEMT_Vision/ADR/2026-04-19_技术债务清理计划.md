# ADR-2026-04-19: 技术债务清理计划

> Architecture Decision Record
> 日期: 2026-04-19
> 决策者: CTO Agent
> 状态: 已接受

---

## 背景

在完成 NEMT 量化系统初始化后，CTO Agent 对代码库进行了技术债务扫描，发现以下关键问题：

1. **安全风险**: Notion API Token 硬编码在代码中
2. **架构偏离**: 使用 http.server 而非架构文档规定的 FastAPI
3. **代码质量问题**: 测试覆盖率不足、配置分散、重复代码

---

## 决策

### 1. 立即处理的安全问题

**问题**: Notion Token 硬编码
- 位置: `notion_theory_sync.py:21`
- 风险等级: 严重

**决策**:
```python
# 改前 (不安全)
NOTION_TOKEN = 'ntn_s640536110656...'

# 改后 (安全)
import os
NOTION_TOKEN = os.getenv("NOTION_TOKEN", "")
```

### 2. API Server 版本合并

**问题**: 存在三个 API Server 版本
- `api_server.py` (v1.1.0)
- `api_server_v2.py`
- `api_server_prod.py`

**决策**: 
- 以 `api_server_v2.py` 为基础合并
- 保留一个统一入口
- 删除废弃版本

### 3. 测试覆盖率目标

**决策**: 采用渐进式达标

| 阶段 | 目标覆盖率 | 时间 |
|------|-----------|------|
| Phase 1 | 50% | 2周 |
| Phase 2 | 70% | 1个月 |
| Phase 3 | 80% | 2个月 |

### 4. API 框架迁移

**问题**: 架构文档规定 FastAPI，实际用 http.server

**决策**:
- 短期: 保持 http.server，确保功能稳定
- 中期: 迁移到 FastAPI (Sprint 3)
- 收益: 类型安全、自动文档、更好的生态

---

## 后果

### 正面
- 提高代码安全性
- 降低维护成本
- 改善长期可扩展性

### 负面
- 需要短期投入时间
- 可能影响开发进度
- 存在迁移风险
