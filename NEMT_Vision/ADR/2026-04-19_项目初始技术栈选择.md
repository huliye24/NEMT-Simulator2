## 背景

NEMT量化操作系统需要选择合适的技术栈来支持：
1. 高性能数值计算（NLS方程求解）
2. 实时市场数据处理
3. 策略回测和模拟
4. Web可视化界面
5. Notion/Obsidian知识管理集成

---

## 候选方案

### 方案A: Python + TypeScript

**核心技术**:
- Python 3.10+ (后端/算法)
- TypeScript + React (前端)
- FastAPI (API框架)
- NumPy/SciPy (数值计算)

**优点**:
- Python量化生态成熟
- TypeScript类型安全
- 团队熟悉度高
- 学习成本低

**缺点**:
- GIL限制多线程性能
- 前端需要单独构建

### 方案B: Rust + TypeScript

**核心技术**:
- Rust (后端/算法)
- TypeScript + React (前端)
- Actix-web (API框架)

**优点**:
- 性能极致
- 内存安全

**缺点**:
- 学习曲线陡峭
- 量化库少
- 开发周期长

### 方案C: Python (全栈)

**核心技术**:
- Python 3.10+
- Flask/Django (Web框架)
- PyQt/Streamlit (UI)

**优点**:
- 单一语言
- 快速开发

**缺点**:
- UI能力弱
- 不适合复杂前端

---

## 决策

**选择**: 方案A - Python + TypeScript

### 决策理由

1. **生态丰富**: Python在量化领域有丰富库（NumPy, SciPy, Pandas）
2. **团队熟悉**: 团队已有Python和TypeScript经验
3. **快速迭代**: 开发效率优先
4. **可扩展**: 性能瓶颈可用Cython/Numba优化

### 技术栈详情

| 层级 | 技术 | 版本 |
|------|------|------|
| 算法核心 | Python | 3.10+ |
| 数值计算 | NumPy | 1.24+ |
| 科学计算 | SciPy | 1.10+ |
| API框架 | FastAPI | 0.100+ |
| 前端框架 | React | 18+ |
| 前端语言 | TypeScript | 5+ |
| 构建工具 | Vite | 5+ |

## 影响

### 正面影响

- 开发速度快
- 社区支持好
- 易于招聘

### 负面影响

- GIL限制（可通过多进程解决）
- 动态类型（TypeScript缓解）

### 缓解措施

- 使用TypeScript保证前端类型安全
- 关键计算使用Numba加速
- 多进程绕过GIL
