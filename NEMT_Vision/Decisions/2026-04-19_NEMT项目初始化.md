## 决策内容

### 1. 创建NEMT_Vision愿景仓库

**决策**: 在项目根目录创建 `NEMT_Vision/` Obsidian仓库

**理由**:
- 愿景、原则需要持久存储和统一访问
- 所有Agent需要只读访问核心文档
- 决策需要有长期记忆

**结构**:
```
NEMT_Vision/
├── VISION.md                 # 核心愿景
├── Principles/               # 详细原则
│   ├── Kitchen_First.md
│   ├── Strategy_As_Organism.md
│   └── Risk_Above_Return.md
├── Decisions/                # 决策记录
├── Boundaries/               # 边界定义
├── Milestones/               # 里程碑记录
└── Templates/               # 模板文件
```

### 2. 确立四大核心原则

**决策**: 确立以下四大核心原则

| 原则 | 核心内容 |
|------|----------|
| 策略是生物体 | 策略有生命周期：出生、成长、衰老、死亡 |
| 系统适应市场 | 不预测市场，而是适应市场变化 |
| 风控优先 | 任何情况下风控优先于收益 |
| 厨房理论 | 架构优先于功能，先建厨房后做菜 |

### 3. 确定V1边界

**决策**: V1阶段明确不做以下事情

| 边界 | 原因 |
|------|------|
| 不做实盘 | V1专注回测验证 |
| 不做Web前端 | 专注后端引擎 |
| 不依赖付费数据 | 降低依赖和成本 |
| 不做高频交易 | 专注日线级别 |
| 不做ML预测 | 坚持规则驱动 |
