## 愿景对齐检查

在做出决策前，逐项检查：

### 使命检查

- [ ] **直接服务**: 这个决策是否直接服务于我们的使命（构建让策略生存、竞争、进化的金融生态系统）？
  - [ ] 是 - 与使命直接相关
  - [ ] 否 - 与使命无直接关系

### 原则检查

**原则一：策略是生物体**
- [ ] 这个决策是否尊重策略的生命周期？
- [ ] 是否有助于策略的"出生-成长-淘汰"机制？

**原则二：系统适应市场**
- [ ] 这个决策是否增强了系统的适应性？
- [ ] 是否避免了试图"预测市场"的陷阱？

**原则三：风控优先**
- [ ] 这个决策是否将风控放在首位？
- [ ] 是否可能引入新的风险？
- [ ] 是否需要额外的风控措施？

**原则四：厨房理论**
- [ ] 底层架构是否稳固到支持这个决策？
- [ ] 是否需要先完善厨房（基础设施）再做这个菜（功能）？

### 边界检查

- [ ] 这个决策是否超出了V1阶段的边界？
  - [ ] 不做实盘交易
  - [ ] 不做Web前端
  - [ ] 不依赖付费数据源
  - [ ] 不追求高频交易
  - [ ] 不做机器学习预测

- [ ] 是否触碰了架构红线？
  - [ ] 不绕过风控模块
  - [ ] 不硬编码策略参数
  - [ ] 支持高可用
  - [ ] 策略需验证

### 简单性检查

- [ ] 有没有更简单的方案？
- [ ] 这个方案是否过度设计？
- [ ] 是否符合"厨房理论"的简单性要求？
