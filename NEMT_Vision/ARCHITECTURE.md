## 七、架构原则

### 7.1 厨房理论

```
┌─────────────────────────────────────────────────────────────────┐
│  ❶ 先搭厨房，再请厨师                                           │
│     → 系统必须先能编译/运行，再谈业务逻辑                        │
│                                                                  │
│  ❷ 厨房阶段禁止深度业务逻辑                                     │
│     → 只有框架、接口、模拟数据                                   │
│                                                                  │
│  ❸ 厨师阶段只填充实现，不改厨房结构                             │
│     → 接口契约神圣不可侵犯                                       │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 模块边界原则

- 每个节点有明确的输入/输出定义
- 节点之间通过接口通信，不直接依赖实现
- Mock实现用于测试和生产切换
