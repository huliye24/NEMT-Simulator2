# 波动率数据预言机设计方案

## 项目概述

利用 NEMT 的波动率分析能力，构建一个差异化的 Switchboard 数据 Feed。

---

## 为什么需要波动率预言机

### 市场空白
```
传统预言机: 喂价格
Pyth/Chainlink: 精准的价格数据 (饱和市场)

波动率预言机: 喂「价格的不确定性」
市场状态: 几乎空白 (高价值蓝海)
```

### 应用场景

#### 1. 期权协议定价
```
问题: 期权需要波动率作为核心输入
现状: 项目方只能使用历史波动率的近似值
解决方案: 实时波动率数据

潜在客户:
- Zeta Markets
- 01 Exchange
- 各种期权协议
```

#### 2. 动态风险管理
```
应用: 清算阈值动态调整
场景:
- 高波动期 → 提高抵押率要求
- 低波动期 → 降低抵押率要求

受益协议:
- 借贷协议 (Solend, Marginfi)
- 保证金交易
```

#### 3. 趋势预测
```
指标: Hurst 指数
应用:
- H > 0.5 → 趋势市场 (顺势策略)
- H < 0.5 → 均值回归 (逆势策略)

使用场景:
- 量化交易机器人
- 套利策略
- 做市商报价
```

## 收益模型

### 方案1: 按使用收费
```
模型: 每查询收取固定费用
定价: $0.001-0.01/次查询
优势: 稳定收入流
预期: 1000-10000 次/天 × $0.005 = $5-50/天
```

### 方案2: 质押收益分享
```
模型: Feed 使用费分给数据提供者
比例: 70% 给提供者, 30% 给协议
优势: 被动收入增长
```

### 方案3: 增值服务
```
模型: 基础 Feed 免费，高级指标收费
定价: $99-499/月
内容: Hurst信号、分形维度、历史回测

目标客户:
- 量化交易团队
- 期权协议
- 借贷协议
```
