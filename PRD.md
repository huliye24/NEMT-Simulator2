# NEMT量化系统 · 产品需求文档 (PRD)

> 版本: v0.1 | 更新: 2026-04-19 | 状态: 🚧 进行中

---

## 1. 产品愿景

**一句话**: 构建一个基于**非平衡市场理论(NEMT)**的智能量化交易系统，让策略在市场中自主生存、竞争、进化。

**核心理念**:
- 不是设计一个"赚钱的策略"，而是设计一个"让好策略脱颖而出的生态系统"
- 策略不是静态的，而是动态适应市场相位的
- 风险不是预设的，而是由市场状态动态感知的

---

## 2. 市场问题

### 2.1 当前市场痛点

| 问题 | 描述 | 影响 |
|------|------|------|
| **市场相位误判** | 大多数策略无法识别当前市场状态，盲目开仓 | 亏损 |
| **静态风控** | 止损/仓位不随市场状态调整 | 过早止损或过度持仓 |
| **链上数据孤岛** | 链上指标与交易策略脱节 | 信号滞后 |
| **策略进化缺失** | 策略无法自我评估和淘汰 | 策略失效后继续亏损 |

### 2.2 NEMT解决方案

- **四相位识别**: 自动识别市场状态(A混乱/B蓄力/C临界/D趋势)
- **相位驱动风控**: 仓位/止损/杠杆随相位动态调整
- **链上+链下融合**: 将MVRV/NUPL等链上指标纳入信号决策
- **策略生态**: 多个策略竞争，优胜劣汰

## 3. 用户故事

### 3.1 核心用户

| 用户 | 角色 | 目标 |
|------|------|------|
| **交易员** | 主策略管理者 | 监控系统状态，批准/否决策略 |
| **量化研究员** | 策略开发者 | 设计、回测、上线新策略 |
| **系统** | 自动执行者 | 根据相位执行交易，无需人工干预 |

### 3.2 用户故事

#### 故事1: 相位识别与自动切换
```
作为 交易员
我想要 系统自动识别市场相位
以便于 我知道当前应该采取什么策略

验收标准:
- [ ] 相位A/B/C/D 能自动识别
- [ ] 相位切换时有明确日志
- [ ] 切换延迟 < 1秒
```

#### 故事2: 动态仓位控制
```
作为 系统
我想要 根据当前相位自动调整仓位
以便于 在高风险期减少暴露，在趋势期最大化收益

验收标准:
- [ ] 相位A: 仓位 ≤ 20%
- [ ] 相位D: 仓位可达 100%
- [ ] 仓位调整有平滑过渡
```

#### 故事3: 策略自动进化
```
作为 量化研究员
我想要 系统自动评估策略表现并淘汰差的
以便于 我只需要设计新策略，不需要手动管理

验收标准:
- [ ] 每周自动生成策略评分
- [ ] 连续2周评分低于阈值自动标记
- [ ] 淘汰决策需要交易员确认
```

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 目标 | 说明 |
|------|------|------|
| **延迟** | < 100ms | 相位识别到信号输出 |
| **吞吐量** | > 1000 信号/秒 | 高频场景支持 |
| **可用性** | 99.9% | 每月停机 < 43分钟 |
| **恢复时间** | < 5分钟 | 故障后自动恢复 |

### 5.2 安全需求

| 需求 | 说明 |
|------|------|
| **数据加密** | 敏感配置加密存储 |
| **API鉴权** | 所有API需认证 |
| **审计日志** | 所有操作可追溯 |
| **隔离运行** | 回测/实盘完全隔离 |
