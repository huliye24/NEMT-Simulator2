## 2. Node Flow详细定义

### 2.1 Market Data Flow

```
数据获取流程
============

Start
  │
  ▼
┌──────────────────┐
│ 接收市场数据请求  │
│ (symbol, interval)│
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 获取K线数据       │◄── Binance API
│ fetch_klines()   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 数据验证          │
│ - 检查数据完整性  │
│ - 检查时间连续性  │
└────────┬─────────┘
         │
    ┌────┴────┐
    │ 验证通过？ │
    └────┬────┘
    是/    \否
    │        │
    ▼        ▼
┌──────┐  ┌──────────────────┐
│ 返回 │  │ 尝试补充数据      │
│ 数据 │  │ 或返回错误        │
└──────┘  └──────────────────┘
```

### 2.2 NEMT Core Flow

```
NEMT核心计算流程
================

K线数据
  │
  ▼
┌──────────────────┐
│ 1. 状态初始化     │
│ initialize_state()│
│ - 提取价格序列    │
│ - 设置初始参数    │
│ - alpha, beta    │
│   noise, dt, dx  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 2. 时间演化       │
│ evolve()         │
│ - Euler方法      │
│ - 步数: steps    │
│ - 返回演化历史    │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 3. 频谱分析       │
│ analyze_spectrum()│
│ - FFT变换        │
│ - 提取谱分量     │
│ - 计算谱宽       │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 4. 共振检测       │
│ detect_resonance()│
│ - 找共振峰       │
│ - 计数           │
│ - 强度评估       │
└────────┬─────────┘
         │
         ▼
返回结果:
{
  "spectralWidth": float,
  "meanFrequency": float,
  "resonancePeaks": [...],
  "evolution": [...]
}
```

### 2.3 Signal Generation Flow

```
信号生成流程
============

NEMT结果 + 市场数据
        │
        ▼
┌──────────────────┐
│ 计算DCI           │
│ direction_consistency │
│ - 趋势方向       │
│ - 一致性评分     │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 涡旋检测 (4条件) │
│ detect_vortex()  │
│ 1. 价格突破      │
│ 2. 谱宽异常      │
│ 3. 频率集中      │
│ 4. 时间窗口      │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 随机共振检测     │
│ detect_resonance()│
│ 1. 噪声匹配      │
│ 2. 非线性增强    │
│ 3. 信号同步      │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 生成综合信号     │
│ generate_signal()│
│ - 置信度计算    │
│ - 信号类型      │
│ - 建议          │
└────────┬─────────┘
         │
         ▼
返回: TradingSignal
{
  "type": "buy"|"sell"|"hold",
  "confidence": 0.0-1.0,
  "indicators": {...}
}
```

### 2.4 Phase State Machine Flow

```
相位状态机流程
==============

输入: NEMT结果, DCI, 市场数据
        │
        ▼
┌──────────────────┐
│ 计算相位指标      │
│ - spectralWidth  │
│ - meanFrequency  │
│ - DCI            │
│ - 价格动量       │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 相位识别         │
│ identify_phase() │
│                  │
│ 规则:           │
│ A (噪声):        │
│   - 谱宽大      │
│   - DCI低       │
│                  │
│ B (涡旋):        │
│   - 谱宽适中    │
│   - DCI上升     │
│                  │
│ C (临界):        │
│   - 共振峰出现  │
│   - DCI>0.6    │
│                  │
│ D (趋势):        │
│   - 谱宽稳定    │
│   - DCI>0.8    │
└────────┬─────────┘
         │
    ┌────┴────┐
    │ 相位变化？ │
    └────┬────┘
    是/    \否
    │        │
    ▼        ▼
┌────────┐  ┌────────┐
│触发事件│  │ 保持   │
│转换    │  │ 当前   │
└────────┘  └────────┘
    │
    ▼
┌──────────────────┐
│ PhaseChangeEvent  │
│ - old_phase      │
│ - new_phase      │
│ - confidence     │
│ - timestamp      │
└──────────────────┘
```

### 2.5 Execution Flow

```
执行层四步流程
==============

信号 + 风控配置
        │
        ▼
┌──────────────────┐
│ STEP 1: 预测     │
│ predict()        │
│                  │
│ - 信号类型       │
│ - 置信度        │
│ - 初步价格目标  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ STEP 2: 入场信号  │
│ generate_entry() │
│                  │
│ - 入场价格       │
│ - 方向          │
│ - 初始仓位      │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ STEP 3: 验证清单  │
│ validate_entry() │
│                  │
│ □ 相位支持       │
│ □ 风险可接受     │
│ □ 无反向信号     │
│ □ 数据新鲜       │
└────────┬─────────┘
         │
    ┌────┴────┐
    │ 全部通过？ │
    └────┬────┘
    是/    \否
    │        │
    ▼        ▼
┌────────┐  ┌──────────────────┐
│ 继续   │  │ 拒绝入场         │
│        │  │ 记录原因         │
└───┬────┘  └──────────────────┘
    │
    ▼
┌──────────────────┐
│ STEP 4: 仓位管理  │
│ add_position()   │
│                  │
│ - 计算仓位       │
│ - 设置止损       │
│ - 设置止盈       │
│ - 更新持仓       │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 离场决策         │
│ manage_exits()   │
│                  │
│ - 止损触发       │
│ - 止盈触发       │
│ - 趋势反转       │
│ - 相位变化       │
└──────────────────┘
```

### 2.6 Risk Management Flow

```
风险管理流程
============

信号 + 当前相位 + 账户状态
            │
            ▼
┌──────────────────┐
│ 计算仓位上限      │
│ calculate_position()│
│                  │
│ 相位配置:        │
│ A: 20%           │
│ B: 50%           │
│ C: 70%           │
│ D: 100%          │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 计算ATR止损       │
│ calculate_stop_loss()│
│                  │
│ 基础止损:        │
│ 涡旋突破: 1.5x  │
│ 随机共振: 2.0x  │
│ 趋势回调: 1.0x  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 计算杠杆          │
│ calculate_leverage()│
│                  │
│ A: 0x (禁止)     │
│ B: 1x            │
│ C: 2x            │
│ D: 1x            │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 检查回撤         │
│ check_drawdown() │
│                  │
│ 绿: < 5%        │
│ 黄: 5-10%       │
│ 橙: 10-20%      │
│ 红: > 20%       │
└────────┬─────────┘
         │
         ▼
返回风控决策
```
