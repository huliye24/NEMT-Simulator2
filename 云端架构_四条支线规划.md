# NEMT系统 · 四条支线规划

> 版本: v0.1 | 更新: 2026-04-19 | 状态: 🚧 规划中

---

## 一、愿景

**通过云服务器作为中枢，让所有计算都走云端，实现：**

```
本地只负责：展示 + 交互 + 轻量控制
云端负责：所有重型计算 + 数据存储 + 策略执行
```

## 三、支线3实现详情（策略层）

### 3.1 已创建的文件

| 文件 | 说明 |
|------|------|
| `quant-sync-server/strategy_service.py` | 策略服务核心实现 |
| `quant-sync-server/routers/strategy_router.py` | 策略API路由 |
| `quant-sync-server/api_server.py` | 更新以集成策略服务 |
| `web/src/services/strategyService.ts` | 前端策略服务 |
| `quant-sync-server/test_strategy_service.py` | 测试脚本 |

### 3.2 策略服务核心功能

**StrategyService 类**：
- `create_strategy(template, name, params)` - 创建策略
- `list_strategies(status)` - 列出策略
- `run_backtest(strategy_id, prices, config)` - 运行回测
- `score_strategy(strategy_id)` - 评分策略
- `evaluate_and_evict()` - 淘汰评估
- `rebalance_weights(mode)` - 权重分配

**StrategyFactory** - 策略模板：
| 模板ID | 名称 | 类型 |
|--------|------|------|
| `trend_ma` | 趋势均线策略 | TREND |
| `mean_reversion_bb` | 布林带均值回归策略 | MEAN_REVERSION |
| `momentum_roc` | 动量ROC策略 | MOMENTUM |
| `nemt_dci` | NEMT DCI策略 | NEMT |
| `nemt_vortex` | NEMT 涡旋策略 | NEMT |
| `hybrid_nemt` | NEMT混合策略 | HYBRID |

### 3.3 API 端点

```
策略管理:
  GET  /api/strategy/list       - 策略列表
  POST /api/strategy/create      - 创建策略
  GET  /api/strategy/get       - 获取策略详情
  PUT  /api/strategy/update     - 更新策略
  DELETE /api/strategy/delete   - 删除策略

回测:
  POST /api/strategy/backtest       - 回测策略
  POST /api/strategy/batch_backtest - 批量回测

评分与淘汰:
  GET  /api/strategy/scores   - 所有评分
  POST /api/strategy/score   - 获取策略评分
  POST /api/strategy/evict   - 触发淘汰评估
  GET  /api/strategy/events  - 淘汰事件

权重:
  POST /api/strategy/rebalance - 重新分配权重

统计:
  GET  /api/strategy/templates - 策略模板
  GET  /api/strategy/stats    - 策略统计
```

### 3.4 使用示例

**创建策略**:
```python
from strategy_service import create_service

service = create_service()
strategy = service.create_strategy("nemt_dci", "我的DCI策略")
```

**运行回测**:
```python
import numpy as np
prices = 67000 + np.cumsum(np.random.randn(500) * 100)
result = service.run_backtest(strategy.id, list(prices))
print(f"收益: {result.return_pct:.2f}%, 夏普: {result.sharpe_ratio:.2f}")
```

**评分与淘汰**:
```python
score = service.score_strategy(strategy.id)
print(f"评分: {score.total_score:.1f}")
service.evaluate_and_evict()
```

### 3.5 前端调用

```typescript
import { getStrategyService } from '../services/strategyService';

const service = getStrategyService();

// 创建策略
const strategy = await service.createStrategy('nemt_dci', '我的策略');

// 运行回测
const result = await service.runBacktest(strategy.id, prices);

// 获取评分
const score = await service.getScore(strategy.id);
```

## 三、支线详细说明

### 支线 ❶：感知层 (Perception Layer)

**目标**：云端统一采集、清洗、存储市场数据

| 模块 | 云端职责 | 本地职责 |
|------|---------|---------|
| 数据采集 | Binance/OKX API → 实时K线 | - |
| 数据清洗 | 异常值过滤 + 缺失填充 | - |
| 数据存储 | SQLite/Redis → 历史数据 | - |
| 链上数据 | MVRV/NUPL/交易所余额计算 | 指标展示 |
| 数据分发 | WebSocket → 实时推送 | 轻量接收 |

**技术选型**：
- 数据采集：`aiohttp` + `websockets`
- 数据存储：`SQLite`（历史）+ `Redis`（实时缓存）
- 数据分发：`FastAPI` WebSocket

**验收标准**：
- [ ] 云端能7x24小时稳定采集数据
- [ ] 断线重连机制正常
- [ ] 本地能实时接收K线更新
- [ ] 历史数据可查询

---

### 支线 ❷：计算层 (Computation Layer)

**目标**：云端执行所有NEMT核心算法

| 模块 | 云端职责 | 本地职责 |
|------|---------|---------|
| NLS求解器 | 非线性薛定谔方程数值求解 | - |
| 四相位检测 | A/B/C/D相位自动识别 | 相位状态展示 |
| 谱分析器 | 频谱计算 + 谱宽分析 | 频谱图展示 |
| 信号生成 | DCI + 涡旋 + 随机共振 | 信号看板 |
| 链上分析 | MVRV/NUPL计算 | 链上指标展示 |

**技术选型**：
- NLS求解：`NumPy` + `SciPy`（或MATLAB引擎）
- 信号计算：`NumPy` 向量化计算
- 分布式计算：`Celery` 任务队列（可选）

**验收标准**：
- [ ] NLS求解 < 100ms
- [ ] 四相位检测准确率 > 85%
- [ ] 信号生成完整输出
- [ ] 支持并行计算多品种

---

### 支线 ❸：策略层 (Strategy Layer)

**目标**：云端管理策略全生命周期

| 模块 | 云端职责 | 本地职责 |
|------|---------|---------|
| 策略工厂 | 策略模板管理 + 参数生成 | 策略列表展示 |
| 回测引擎 | 历史数据回测 + 性能计算 | 回测报告展示 |
| 评分系统 | Sharpe/回撤/胜率计算 | 评分排名展示 |
| 淘汰机制 | 差策略自动标记/休眠 | 淘汰记录展示 |
| Notion同步 | 策略双向同步 | 状态同步 |

**技术选型**：
- 回测引擎：事件驱动循环（自建）
- 策略存储：SQLite + Notion备份
- 任务调度：`APScheduler`

**验收标准**：
- [ ] 支持10+策略同时回测
- [ ] 回测速度：1年数据 < 10秒
- [ ] 策略评分自动更新
- [ ] Notion策略同步正常

---

### 支线 ❹：执行层 (Execution Layer)

**目标**：云端执行交易 + 实时风控

| 模块 | 云端职责 | 本地职责 |
|------|---------|---------|
| 订单路由 | 信号 → 订单转换 | - |
| 交易所对接 | Binance/OKX API下单 | - |
| 仓位管理 | 相位驱动仓位调整 | 仓位状态展示 |
| 风险控制 | ATR止损/回撤控制 | 风控告警展示 |
| 交易记录 | 完整交易日志 | 交易历史展示 |

**技术选型**：
- 交易所对接：`ccxt` 库
- 风控计算：实时ATR + 回撤监控
- 交易记录：SQLite + 文件日志

**验收标准**：
- [ ] 订单延迟 < 500ms
- [ ] 风控规则触发准确
- [ ] 交易记录完整可查
- [ ] 紧急止损自动执行
